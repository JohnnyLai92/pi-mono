# mock.md — 外部依賴模擬與錯誤處理策略（Mock & Fallback）

## 適用範圍

本文件定義系統依賴的外部服務清單，以及各服務失敗時 Agent 的模擬回傳與處理策略。

---

## 1. 外部服務依賴清單

| 服務 | 用途 | 等級 | 失敗影響 |
|------|------|------|---------|
| GitHub API | git push / pull / 同步程式碼 | 🔴 關鍵 | 無法上傳程式碼至遠端 |

> 等級說明：🔴 關鍵（掛了無法繼續該操作）/ 🟡 非關鍵（掛了可跳過或替代）

---

## 2. 全局錯誤處理策略

所有外部服務失敗時，統一遵守以下流程：

```
第 1 次失敗 → 等待 5 秒 → 自動重試
第 2 次失敗 → 等待 10 秒 → 自動重試
第 3 次失敗 → 等待 20 秒 → 自動重試
第 4 次失敗 → 等待 30 秒 → 自動重試
第 5 次失敗 → 停止執行 → 向上級回報錯誤
```

### 重試參數

| 參數 | 值 |
|------|-----|
| 最大重試次數 | 5 次 |
| 重試間隔策略 | 遞增等待（5s → 10s → 20s → 30s → 30s） |
| 重試後仍失敗 | 停止執行，回報錯誤 |

---

## 3. 各服務錯誤處理細則

### 3.1 GitHub API

| 錯誤碼 | 說明 | 處理方式 |
|--------|------|---------|
| 401 Unauthorized | Token 無效或過期 | 停止重試 → 立即回報「GitHub 認證失敗，請檢查 Token」 |
| 403 Forbidden | 權限不足或速率限制 | 執行重試策略（5 次） |
| 404 Not Found | 倉庫或分支不存在 | 停止重試 → 立即回報「目標倉庫或分支不存在」 |
| 408 Timeout | 請求逾時 | 執行重試策略（5 次） |
| 409 Conflict | 合併衝突 | 停止重試 → 回報衝突細節，等待強尼指示 |
| 422 Unprocessable | 請求參數錯誤 | 停止重試 → 回報錯誤參數 |
| 500 / 502 / 503 | GitHub 伺服器異常 | 執行重試策略（5 次） |

### 不重試的情境（立即回報）

以下錯誤重試無意義，應立即停止並回報：

- 401 認證失敗
- 404 資源不存在
- 409 合併衝突
- 422 參數錯誤

---

## 4. 錯誤回報格式

重試全部失敗後，依以下格式回報：

```
[時間][Agent][外部服務錯誤回報]
├── 服務：GitHub API
├── 操作：git push origin main
├── 錯誤碼：503 Service Unavailable
├── 重試次數：5 / 5（已達上限）
├── 原始錯誤：The service is temporarily unavailable. Please retry later.
├── 影響範圍：程式碼未同步至遠端倉庫
└── 建議：等待 GitHub 服務恢復後重新執行
```

---

## 5. 模擬回傳值（測試用）

在測試環境中，可使用以下模擬值替代真實 API 回應：

### GitHub API — 正常回傳

```json
{
  "status": 200,
  "message": "OK",
  "data": {
    "commit": "abc1234",
    "branch": "main",
    "pushed": true
  }
}
```

### GitHub API — 失敗回傳

```json
{
  "status": 503,
  "message": "Service Unavailable",
  "data": null
}
```

## 6. 擴充規則

未來若新增其他外部服務依賴（如 LINE API、HuggingFace、LLM API 等），依以下步驟擴充本文件：

1. 將服務加入第 1 節的依賴清單，標註等級
2. 在第 3 節新增該服務的錯誤處理細則
3. 在第 5 節新增該服務的模擬回傳值
4. 若遇到需要特別處理的錯誤情境，與強尼確認後補充
