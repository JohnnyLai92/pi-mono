# memory.md — 上下文記憶管理策略（Context System）

## 適用範圍

本策略適用於系統中所有 Agent（大寶、小寶），規範記憶的來源、優先順序、儲存格式與提取規則。

---

## 1. 記憶讀取優先順序

Agent 在處理任務時，依以下順序提取上下文。高優先順序的資訊覆蓋低優先順序的衝突內容：

```
優先順序 1（最高）：強尼的即時指令
  │  當次對話中強尼直接下達的命令與需求
  ▼
優先順序 2：專案設定檔
  │  工作目錄下的 AGENTS.md、package.json、tsconfig.json 等
  ▼
優先順序 3：歷史對話紀錄
  │  同一會話中的先前對話內容與任務結果
  ▼
優先順序 4（最低）：通用知識
     Agent 自身的訓練知識與一般常識
```

### 衝突處理

| 情境 | 處理方式 |
|------|---------|
| 即時指令與專案設定檔矛盾 | 以即時指令為準 |
| 即時指令與 rule.md 禁止行為矛盾 | 以 rule.md 為準（安全優先），主動詢問強尼 |
| 歷史紀錄與即時指令矛盾 | 以即時指令為準（視為需求變更） |

## 2. 記憶來源

### 2.1 常駐記憶（每次會話自動載入）

| 來源 | 路徑 | 說明 |
|------|------|------|
| 個人偏好 | `profile.md` | 強尼的角色設定、飲食習慣、健康目標、已整合功能 |
| 行為護欄 | `rule.md` | 全局禁止行為與操作確認規則 |
| Agent 定義 | `agents.md` | 大寶、小寶的職責與權限 |

### 2.2 專案記憶（進入工作目錄時載入）

| 來源 | 說明 |
|------|------|
| `AGENTS.md` | 該專案的開發規範與指引 |
| `package.json` | 套件版本、scripts、依賴關係 |
| `tsconfig.json` | TypeScript 編譯設定 |
| `README.md` | 專案說明與架構概覽 |

### 2.3 會話記憶（當次對話中累積）

| 來源 | 說明 |
|------|------|
| 強尼的即時指令 | 當次對話中的所有命令 |
| 任務執行紀錄 | 大寶與小寶的執行過程與結果 |

---

## 3. 記憶儲存格式

所有任務執行紀錄統一使用以下格式：

```
[時間][Agent][事件][結果]
```

### 欄位定義

| 欄位 | 格式 | 說明 |
|------|------|------|
| 時間 | `YYYY-MM-DD HH:MM` | 事件發生的時間點 |
| Agent | `大寶` / `小寶` | 執行該事件的 Agent |
| 事件 | 自由文字 | 執行了什麼操作 |
| 結果 | `✅ 成功` / `❌ 失敗` / `⏳ 進行中` | 執行結果與摘要 |

### 範例

```
[2026-04-06 14:30][大寶][接收命令：重啟 LineBot 服務][✅ 成功：已分派給小寶]
[2026-04-06 14:31][小寶][終止舊 gunicorn 程序 PID 68394][✅ 成功：程序已停止]
[2026-04-06 14:31][小寶][啟動新 gunicorn 程序][✅ 成功：新 PID 8807，port 5000]
[2026-04-06 14:32][小寶][向大寶回報完成結果][✅ 成功：服務已重啟]
[2026-04-06 14:32][大寶][檢查服務狀態][✅ 成功：HTTP 200 回應正常]
[2026-04-06 14:32][大寶][向強尼回報最終結果][✅ 成功：LineBot 服務重啟完成]
```

## 4. 記憶提取規則

### 4.1 Agent 啟動時

```
1. pi_startup.py 掃描 ~/.pi/memory/*.md → 寫入 LTM section
2. pi_startup.py 掃描 .pi/short_term_memory/*.md → 寫入 STM section
3. 兩個 section 合併寫入 .pi/APPEND_SYSTEM.md（agent 啟動時自動載入）
4. 若存在未彙整的跨日 STM → 顯示 startup 提醒
5. 載入工作目錄專案記憶（AGENTS.md 等）
```

### 4.2 任務執行中

```
1. 優先查找即時指令中的相關上下文
2. 若即時指令不足 → 查找專案設定檔
3. 若專案設定檔無相關資訊 → 查找歷史對話
4. 以上皆無 → 使用通用知識，並標註「基於通用知識判斷」
```

### 4.3 資訊不足時

| 情境 | 處理方式 |
|------|---------|
| 記憶中找不到相關上下文 | 主動詢問強尼（遵守 rule.md 第 3 條） |
| 多筆記憶互相矛盾 | 依優先順序取高優先者，並告知強尼存在矛盾 |
| 記憶內容可能已過時 | 標註「此資訊可能已過時」，詢問是否需要更新 |
