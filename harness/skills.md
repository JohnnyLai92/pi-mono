## SK-01：Director_Clear

| 欄位 | 內容 |
|------|------|
| 名稱 | `Director_Clear` |
| 用途 | 分析指定目錄的內容，辨識可能可刪除的檔案，輸出分析報告 |
| 執行者 | 小寶 |
| 審批 | `[Requires_Approval]` — 分析報告產出後，需經強尼確認才可執行刪除 |

### 參數

| 參數 | 型別 | 必填 | 說明 |
|------|------|------|------|
| `directory_path` | string | ✅ | 要分析的目錄路徑 |

### 執行流程

```
1. 大寶接收強尼的指令，取得目標目錄路徑
2. 大寶分派 Director_Clear 任務給小寶
3. 小寶掃描目錄，分析內容：
   ├── 列出所有檔案與子目錄
   ├── 標記檔案類型（原始碼 / 設定檔 / 快取 / log / 暫存 / 其他）
   ├── 標記最後修改時間
   ├── 辨識可能可刪除的檔案：
   │     ├── __pycache__ / .pyc
   │     ├── node_modules（可重新安裝）
   │     ├── dist / build（可重新編譯）
   │     ├── *.log（歷史 log）
   │     ├── .DS_Store
   │     ├── 暫存檔 / 備份檔（*.bak, *.tmp, *.swp）
   │     └── 其他重複或過時的檔案
   └── 計算各分類佔用空間
4. 小寶產出分析報告，向大寶回報
5. 大寶檢查報告，向強尼呈現
6. [Requires_Approval] 強尼確認要刪除的項目後，才執行刪除
```

### 分析報告格式

```
# 目錄整理分析報告

**目標目錄**：<directory_path>
**掃描時間**：<YYYY-MM-DD HH:MM>
**總檔案數**：<N> 個
**總佔用空間**：<size>

## 檔案分類統計

| 分類 | 檔案數 | 佔用空間 |
|------|--------|---------|
| 原始碼 | X | XX MB |
| 設定檔 | X | XX MB |
| 快取 / 編譯產物 | X | XX MB |
| Log 檔 | X | XX MB |
| 暫存 / 備份 | X | XX MB |
| 其他 | X | XX MB |

## 建議可刪除的檔案

| # | 檔案路徑 | 分類 | 大小 | 最後修改 | 刪除理由 |
|---|---------|------|------|---------|---------|
| 1 | ./__pycache__/ | 快取 | 2MB | 2026-04-01 | Python 快取，可自動重建 |
| 2 | ./logs/old.log | Log | 15MB | 2026-03-15 | 過時 log |
| ... | | | | | |

**預估可釋放空間**：<size>

## ⚠️ 等待確認

請確認要刪除的項目編號（例如：`1,2,3` 或 `全部` 或 `略過`）。
```

### 限制與護欄

| 規則 | 說明 |
|------|------|
| ⛔ 僅限指定目錄 | 只能掃描與操作 `directory_path` 範圍內的檔案，禁止存取上層或外部路徑 |
| ⛔ 禁止自動刪除 | 分析報告產出後，**必須等待強尼確認**才可執行刪除 |
| ⛔ 禁止刪除原始碼 | 即使強尼確認，若目標為 .ts / .py / .js 等原始碼檔案，需**再次警告**確認 |
| ⛔ 路徑驗證 | 執行前驗證 `directory_path` 確實存在且在工作目錄範圍內 |
