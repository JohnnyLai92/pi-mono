# 對話紀錄與記憶彙整管理 (Conversation Memory Management)

## 目標
透過記錄每日對話並定期彙整，將對話中產生的重要資訊（偏好、設定、任務）轉化為長期記憶，避免遺漏強尼的需求。

## 執行規則
- **對話記錄**：
  - 每次對話結束或有重要決策時，將重點記錄於 `logs/sessions/YYYY-MM-DD.md`。
  - 記錄內容包含：討論主題、強尼的決定、小白承諾的執行事項。
- **主動彙整時間**：每天 17:00 (Asia/Taipei)。
- **彙整流程**：
  1. 讀取前一日的 `logs/sessions/` 記錄。
  2. 分析對話內容，提取出「潛在的長期記憶項目」。
  3. 產出彙整清單，並主動詢問強尼：「強尼，根據昨天的對話，我發現以下幾點可能需要加入長期記憶，請問是否更新？」
  4. 經強尼確認後，更新至對應的記憶檔案（如 `profile.md`, `preferences.md` 或建立新檔案）。
