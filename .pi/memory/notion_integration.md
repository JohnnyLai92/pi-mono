# Notion 整合與調用規則

## 目標
確保小白在需要調用 Notion 資料或頁面時，能正確獲取授權資訊並執行對應 Skill。

## 執行規則
- **憑證獲取**：每次在調用 Notion API 或執行相關 Skill 前，必須確保從 `.env` 環境變數中取得以下資訊：
  - `NOTION_TOKEN`：Notion API Integration Token
  - `NOTION_DATABASE_ID`：目標資料庫的唯一識別碼
- **資料讀取流程**：
  1. 檢查環境變數是否存在。
  2. 使用 `packages/mom/src/skills/notion_reader.py` 中的 `NotionSkill` 類別。
  3. **當強尼要求「列出未完成工作項目」或執行每小時巡檢時** $\rightarrow$ 必須到 Notion 中名為 **「小白待處理工作」** 的資料庫中進行搜尋。
  4. 呼叫 `get_unfinished_tasks()` 方法獲取名單。
  5. **心跳記錄**：在執行上述巡檢後，必須在 **「小白的工作清單」** 資料庫的 `說明` 屬性中增加：`上次小白登入時間：[YYYY-MM-DD HH:MM:SS]`。
- **練習題管理 (V2.0 - 記憶強化版)**：
  - **發佈分流**：
    - Email：僅發送「題目已更新」提醒，**絕不預先提供答案**。
    - Notion：在 `AI 中級應用規劃師題型練習` 建立子頁面（題目），狀態屬性設為 `待作答`。
  - **巡檢與批改 (每小時)**：
    - 僅處理狀態為 `送出批改` 的頁面 $\rightarrow$ 批改答案 $\rightarrow$ 更新狀態為 `已批改` $\rightarrow$ 補上正解與解析。
    - **後續反饋**：批改完成後，立即寄發【完整解析 Email】。
  - **弱點追蹤**：
    - 答錯題目 $\rightarrow$ 記錄至弱點資料庫 $\rightarrow$ 觸發間隔重複 (T+1, T+3, T+7) 重測排程。
- **狀態判定**：- 凡是狀態不屬於 `["完成", "Done", "Completed", "已完成"]` 的項目均視為「未完成」。

## 操作經驗紀錄 (Skill Patterns)
- **讀取頁面**：透過 `/blocks/{page_id}/children` API 遞迴讀取區塊內容。
- **清除內容**：透過將區塊 `archived` 設為 `True` 來刪除特定標題下的所有-項目。
